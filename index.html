Terraform & HCL History

The evolution of Infrastructure as Code

2014

HashiCorp Configuration Language (HCL) Born

HCL was created as a human-readable configuration language specifically designed for DevOps tools and infrastructure automation.

# Early HCL syntax example
variable "region" {
  default = "us-west-2"
}
2014

Terraform Initial Release

Terraform v0.1 was released, introducing the concept of infrastructure as code with a declarative configuration language.

  • Declarative configuration syntax
  • Resource graph building
  • Plan and apply workflow
2016

Terraform 0.7 - Major Enhancements

Introduced modules, remote state, and improved provider ecosystem with AWS, Azure, and Google Cloud support.

terraform init - Initialize Terraform configuration
2019

Terraform 0.12 - HCL2 Revolution

Major language improvements including first-class expressions, rich type system, and template directives.

# HCL2 with for expressions
locals {
  instance_names = [for i in range(3) : "server-${i}"]
}
2021

Terraform Cloud & Enterprise

Expanded collaboration features, policy as code with Sentinel, and enhanced security controls.
